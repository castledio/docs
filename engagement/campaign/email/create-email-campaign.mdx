title: "Create Campaign"
---

### 1. Initiate campaign creation.  

Go to List Campaign page on the menu and click CREATE CAMPAIGN button.
![](/images/email-campaign-1.png)

### 2. Select the email channel and the required trigger type.

Select the Email channel and pick the trigger type. Castled supports the following trigger types for email :  
1. [One time]() - Sends one time email message now or at a scheduled time.
2. [Event triggered]() - Sends one time email based on user's action in the app.

![](/images/email-campaign-2.png)

### 3. Select the User segment. 

Enter the campaign name and select the User segment on which the email campaign needs to be run.

![](/images/email-campaign-3.png)

![](/images/email-campaign-4.png)

### 4. Select the email template to be used.

Select the required email template to be used for the email campaign. Email template creation is covered in detail [here](/engagement/message-template/email/create-template).

![](/images/email-campaign-5.png)

### 5. Capture the Sender details.

In this section the following details need to be captured
1. **Email subject**
2. **Preheader Text** 
3. **Sender Name**
4. **Sender email address**
5. **Reply to email address**

Email subject and preheader text can be [personalised](/engagement/personalisation/personalisation) by typing @.

![](/images/email-campaign-6.png)

![](/images/email-campaign-7.png)

### 6.Modify/edit the email content.

Content details are preloaded based on the template selected. Marketers can modify the email content to meet the campaign requirements along with adding personalised content.

![](/images/email-campaign-8.png)

### 7. Select the physical address to be used in the campaign.

Marketers can select the physical address to be used in the campaign from the list of prepopulated physical address. If the physical address is missing in the dropdown, it can be added in the settings page by referring [here](/settings/physical-address/create-address).

![](/images/email-campaign-9.png)

### 8. Send test email.

It is important to test the campaign email before sending it out to a broader audience. Castled provides the option to send a test email to a list of emails before sending the campaign.  

The following details needs to be captured in the Send test email popup :
1. Test subject
2. Sender Email address
3. Recipient Email addresses - more than one recipient email address can be added comma seperated for sending the test email.

![](/images/email-campaign-10.png)

![](/images/email-campaign-11.png)

### 9. Click NEXT to schedule the campaign.

![](/images/email-campaign-12.png)

### 10. Select the campaign schedule.

Castled supports the following schedule for email campaigns
1. Send immediately
2. Send at a future date/time.

![](/images/email-campaign-13.png)

### 13. Publish the campaign.   

Once all the campaign steps are completed, the campaign can be published by clicking the PUBLISH button.

![](/images/email-campaign-16.png)

### 14. List of the published campaigns.  

Successfully created campaigns are listed in the Campaign list screen. The status of the campaigns will be one of the follow based on the scenario.
1. **RUNNING** - When the published campaign is active and is currently running.
2. **SCHEDULED** - When the published campaign is scheduled for a future date.
3. **COMPLETED** - When the campaign run is complete.
4. **FAILED** - When the campaign run fails for some reason. Failure reason is displayed to aid in figuring out the reason and.
